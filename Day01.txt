1. HTML이란
     - Hyper Text Markup Language
        하이퍼 텍스트를 마크업하는 언어
     - 하이퍼텍스트 : 웹 사이트에서 링크를 클릭하면 다른 문서나 다른 사이트로 즉시 이동이 되는 기능
     - 마크업 : HTML태그들을 이용해서 HTML문서의 body부분을 만들어 줌
     - 마크업을 진행할 때는 약속된 표기법을 이용한다
        <, > 로 표시된 태그들을 이용
     - 대부분의 태그들은 여는 태그와 닫는 태그가 쌍으로 존재
        <h1>~</h1>, <p>~</p>
     - 몇개의 태그들은 닫는 태그가 존재하지 않음
        <img>, <br>...
     - 태그들은 각각의 속성이 존재해서 속성 값들과 함께 사용해야 함
        <img src="이미지의 경로" width="이미지 가로 길이" height="이미지 세로 길이">
     - 태그안에 다른 태그를 포함할 때는 포함관계를 명확하게 표시해야 함
        <div>
            <a href="#"></a>
        </div>
     - ! + tab = html문서
     - 웹 문서에서는 특수문자나 특수기호를 입력
      HTML특성상 공백이 제대로 표시가 안되기 때문에 특수기호를 이용
      <, > 는 태그로 인식되기 때문에 특수기호로 표시
      공백 한 칸 : &nbsp;
      < : &lt;
      > : &gt;
      " : &quot;

2. 텍스트 관련 태그
    - <h1>~<h6> : 제목을 표시해 주는 태그
                  h1이 가장 크고 진하게 표시됨
                  순서대로 글자가 작아지고 연하게 표시됨
    - <p> : 단락을 만들어주는 태그
            텍스트를 입력 시 가장 많이 이용되는 태그
            단락은 앞 뒤에 줄바꿈이 있는 구문
            </p>를 만나기 전까지는 한줄로 표시, 브라우저 사이즈에 따라서 자동으로 줄바꿈
    - <br> : 줄 바꿈 태그
            <p>안에서 enter키를 입력해도 줄바꿈이 되지 않기 때문에 <br>태그를 이용해서 줄 바꿈
    - <hr> : 수평줄을 표시하는 태그
             주제가 바뀌거나 분위기 전환 시 많이 사용되고
             CSS로 스타일 변경 가능
    - <pre> : 입력하는 내용 그대로 화면에 표출 됨
               pre태그 내에서는 공백이나 줄바꿈이 그대로 표출 됨
    - <span> : 줄 바꿈없이 단락을 묶어주는 태그
               span태그 자체로는 의미가 없지만, 특정부분에만 스타일을 적용할 때 사용

3. 목록을 만들어 주는 태그
    - <ul> : 순서없는 목록 만들기
             ul : unorder list의 약자
             ul태그안에 각 항목을 입력할 떄는 <li> 사용
             ul의 li태그에는 작은 원이나 사각형같은 불릿이 생성 됨
             CSS의 list-style-type 속성으로 불릿 모양 변경
    - <ol> : 순서잇는 목록 만들기
             ol : order list의 약자
             ol태그도 각 항목은 li태그 사용
             ol태그는 목록의 숫자표기 방법이나 시작하는 숫자 직접 선택 가능
             type 속성 : 숫자 표기 방법 지정
             start 속성 : 시작 숫자 설정
             reserved 속성 : 항목 역순 표시
    - </li>의 생략
      </li>를 생략해도 다음<li>를 만나면 </li>가 있는 것 처럼 인식 but 시인성이 좋지않음
    - 중첩 목록 만들기
      <li></li> 안에 ul, ol 태그를 생성해서 중첩 목록 만듬

4. 표를 만드는 태그
    - <table></table> : 표의 위치 지정 태그
    - <tr> : 표에 행을 추가해주는 태그
    - <th> : 행의 제목이 되는 태그
    - <td> : 행의 내용이 되는 태그
    - colspan, rowspan 속성 : 지정한 셀 개수 셀 병합
      colspan : 지정한 셀 개수만큼 가로로 합치기
      rowspan : 지정한 셀 개수만큼 세로로 합치기
    - 표의 구조 정의
    - <thead> : 표의 제목 부분
    - <tbody> : 표의 본문 부분
    - <tfoot> : 표의 요약 부분
    https://www.tablesgenerator.com/html_tables

5. 이미지
    - 웹에서 주로 사용하는 이미지 형식
      GIF : 움직이는 이미지 파일
            최대 색상 수 256.
            파일 크기가 작아서 아이콘이나 불릿 등으로 사용
      JPG/JPEG : GIF 파일보다 다양한 색상이나 명암 표현 가능
                  저장을 여러번 반복하면 화질이 저하 됨
      PNG : 투명 배경 이미지
            로고 제작 시 주로 사용
      SVG : 코드로 된 벡터 이미지 파일
            확대하거나 축소해도 이미지가 깨지지 않고
            반복 저장해도 화질의 저하가 없음
    - <img> : 이미지를 삽입하는 태그
               src 속성 : 이미지의 경로 지정
                          이미지의 경로는 현재 웹문서의 위치를 기준으로 지정
                          src="images/road.jpg" => 현재 웹 문서가 있는 폴더의 images폴더의 road.jpg파일
                          src="../images/road.jpg" => 현재 웹 문서가 있는 폴더의 상위폴터에 있는 ~
               alt 속성 : 시각 장애인들을 위한 대체 텍스트 속성
               width, height 속성 : 이미지의 크기 지정하기

6. 하이퍼링크
    - <a> : 클릭 시 다른 페이지로 이동하는 태그
    - href 속성 : 이동할 페이지 지정
    - target 속성 : 이동할 페이지를 표시할 위치(현재 창, 새창) 지정
    - a태그 생성시 밑줄이 쳐진 파란색 텍스트로 만들어지는데, CSS로 색상 및 밑줄을 변경할 수 있음

7. 사용자의 입력 값을 받는 <form>
    - 회원가입이나 로그인 등 사용자의 입력 값을 서버로 전송 할 때 주로 사용
    - <form>~</form>안에 사용자의 입력을 받는 input, date, checkbox 등등 입력 폼을 만들 수 있다
    - form 에서 사용하는 속성들
      method 속성 : 사용자의 입력 값을 서버로 전송하는 방식을 지정
                     get - 주소표시줄에 사용자 입력 값이 표시됨
                           보안상 좋지 않음(주소표시줄에 키값이 노출되기 때문)
                           다른 페이지로 이동할 때 노출되도 되는 파라미터 전송
                           현재 페이지 번호나 현재 카테고리 등
                     post - 사용자의 입력 값이 주소표시줄에 노출되지 않아서 보안에 용이
                           회원가입, 로그인 등 사용자 정보가 입력될 때 사용
      name 속성 : 폼의 이름을 지정
                  한 웹문서 안에 여러개의 form태그가 존재할 때 구분하기 위해 사용
      action 속성 : form 태그 아에 내용들을 처리해줄 서버 프로그램 지정
                     자바에 메소드를 만들고(@requestMapping("메소드의 이름"))
                     action 속성으로 메소드의 이름 호출 -> 자바의 메소드 실행

8. form요소에 입력 값을 알려주는 <label>
    - 사용자가 어떤 값을 입력해야 되는지 텍스트로 표시
    - label 지정 방식1
      <label>아이디
         <input type="text">
      </label>
    - label 지정 방식2
      <label for="user_id">아이디</label>
      <input type="text" id="user_id">
      for 속성을 이용하여 입력 요소를 지정
   
9. 라디오버튼과 체크박스에서의 <label>
    - label 태그 없이 라디오버튼이나 체크박스를 만들면
      만들어진 라디오버튼과 체크박스를 직접 클릭해줘야됨
    - label 태그를 사용하면 label에서 지정한 텍스트를 클릭해도
      라디오버튼이나 체크박스가 선택되도록 할 수 있음
      **라디오 버튼에서 name속성을 동일하게 주면 택1이 되게 됨

10. form요소를 그룹으로 묶어주는 fieldset, legend 태그
    - 쇼핑몰 사이트 주문 폼의 배송정보와 주문정보로 나눠지듯 폼 요소들을 그룹으로 관리
    - fieldset 태그는 폼 요소를 그룹으로 묶어주고
      묵인 그룹에 외곽선 하나 생성
    - legend 태그는 fieldset으로 묶인 그룹의 제목을 지정

11. 사용자가 입력하는 공간 <input>
    - input은 type에 따라 아이디, 이름, 비밀번호, 주소, 전화번호, 날짜, 시간 등 매우 다양한 값을 받을 수 있음
    - id 속성 : type이 동일한 input 태그가 여러번 사용될 때 각각을 구분해주기 위해 사용 됨
    - name 속성 : type이 동일한 input 태그가 여러번 사용될 때 각각을 구분해주기 위해 사용 됨
                  id 속성과의 차이는 form요소가 서버로 전송되거나
                  다른 페이지로 이동할 때 name속성을 키 값으로 가지게 됨
    - type 속성 : input이 어떤 입력 값을 받을 지 지정
                  type="hidden" : 사용자에게는 표출안되고 서버로 넘겨지는 값을 가지고 있을 수 있음
                  *type="text" : 한 줄짜리 텍스트 입력 상자
                  type="search" : 검색 상자
                  type="tel" : 전화번호 입력 상자
                  type="url" : URL 주소 입력 상자
                  type="email" : 메일 주소 입력 상자
                  *type="password" : 비밀번호 입력 상자(입력값이 *로 표시됨)
                  type="datetime" : 국제 표준시 기준으로 설정된 날짜와 시간
                  type="datetime-local" : 사용자 지역 기준으로 설정된 날짜와 시간 입력상자
                  type="date" : 사용자 지역 기준으로 설정된 날짜(년, 월, 일)
                  type="month" : 사용자 지역 기준으로 설정된 날짜(년, 월) 입력 상자
                  type="week" : 사용자 지역 기준으로 설정된 날짜(년, 주) 입력 상자
                  type="time" : 사용자 지역 기준으로 설정된 시간(시, 분, 초, 분할 초) 입력 상자
                  type="number" : 숫자를 조절할 수 있는 화살표 상자
                  type="range" : 숫자를 조절할 수 있는 슬라이드 상자
                  type="color" : 색상 표 선택 상자
                  *type="checkbox" : 주어진 항목에서 2개 이상 선택 가능한 체크박스 상자
                  *type="radio" : 주어진 항목에서 1개만 선택 가능한 라디오 버튼 상자
                  type="file" : 파일을 첨부할 수 있는 버튼
                  type="submit" : action 속성으로 지정된 서버 프로그램 호출
                  type="reset" : 폼 요소를 초기화하는 리셋 버튼
                  *type="button" : 아무기능 없는 버튼

12. type="radio", type="checkbox" 사용 가능한 속성
    - name 속성 : 라디오 버튼이나 체크박스가 여러개 있을 때
                  구분하기 위한 이름 지정
                  라디오 버튼은 주어진 항목에서 하나만 선택 가능하므로
                  같은 항목은 같은 name값으로 지정
    - value 속성 : 선택된 체크박스나 라디오버튼이 서버로 넘어갈 때 넘겨줄 값
    - checked 속성 : 라디오 버튼이나 체크박스의 항목들은 초기에 아무것도 선택되지 않은 상태로 표시되는데,
                     초기에 선택 표시를 하고 싶은 항목을 지정할 때 사용

13. input 태그의 다양한 속성
    - autofocus 속성 : 초기 페이지 로딩 시 원하는 요소에 마우스 커서를 표시
    - placeholder 속성 : 사용자 입력 내용에 대한 힌트를 입력
    - readonly 속성 : 읽기 전용 요소로 만들어 주는 속성
    - required 속성 : 필수 입력 요소로 지정
